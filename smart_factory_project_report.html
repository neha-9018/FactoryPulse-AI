Smart Factory Industry 4.0 Dashboard

Centralized Monitoring System Fusing SCADA Telemetry, ML Failure Forecasting, and CV Quality Audits

1. Project Overview & Objective

The objective of this project is to develop a centralized Industry 4.0 analytics platform that automates shop floor control and provides visibility into operational efficiency. By combining IoT sensor pipelines, Machine Learning algorithms, and Computer Vision inspections, the platform minimizes unplanned downtime and optimizes manufacturing yields.

Key Project Pillars:
  • Predictive Maintenance (ML): Real-time forecasting of mechanical wear anomalies.
  • Vision-Based Inspection (CV): Spindle parts error check scans.
  • SCADA Auto-Stop overrides: Safety shut-downs mapped directly to PLC endpoints.

2. Machine Learning Architecture (ML Pipeline)

The predictive diagnostics pipeline continuously monitors physical parameters like temperature, current draw, and vibration. Telemetry datasets are normalized using standard scalers and processed by an ML classifier.

A. Dynamic Health Index Formula

The health score of each machine is calculated dynamically in the backend using the following weighted linear formulation:

Health Score = 40% (Vibration) + 30% (Temperature) + 20% (Current) + 10% (Reject Rate)

B. Health Status Boundaries

  • Healthy (>80%): Safe operational levels; standard schedules active.
  • Warning (60% - 80%): Minor wear anomalies; inspection advised.
  • Critical (<60%): High-risk operational failure; auto-stop triggered.

3. Computer Vision Inspection

The conveyor check station utilizes grayscaled camera checks and boundary pixel mapping models to identify parts status instantly.

  • LABEL_MISSING: Fails boundary checks for sticker packaging.
  • SURFACE_CRACK: OpenCV contour edge checks.
  • DAMAGE: Discrepancies in target surface area bounds.
  • WRONG_COLOR / DIMENSION: Dimension deviations.

5. Auto-Stop SCADA Request Triggers

The dashboard implements automated safe stop requests to SCADA under two conditions:

  1. Critical Health Drop: If the calculated ML health score falls below 60%, the backend triggers a safe shutdown override, setting status to OFFLINE and dispatching a PLC Stop alert.
  2. Defects Rate Override: If 3 consecutive products fail the OpenCV inspections, the PLC auto-stops the conveyor spindle line to inspect the machine clamps.
